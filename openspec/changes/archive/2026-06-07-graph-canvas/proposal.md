## Why

特性 7 的外壳把一个种子节点交进画布区域就停下了——它只解决「从哪开始」。但探索的
本质是「往下走」:看到一个节点,点开它,看见它的邻居,再点开邻居。没有这一层,种子
就是一个孤零零、点不动的图标,/expand 也无人调用。本能力提供这一层:把种子渲染成
图,并让每一次点击把图长大一圈。

图是增量生长的,不是一次拉全。每次点击只展开一个节点的直接邻居(经 /expand)。这意味
着同一个节点、同一条边可能从不同路径被反复带出来,画布必须去重——已在图中的节点不重
复添加,已存在的边不重复连。否则反复点击会让图爆出重复元素。

每一次展开都是一次独立的异步操作,可能在进行中、可能失败。展开态必须落在「被点的那个
节点」上,而不是整张画布——用户点 A 时,B 的展开失败不该影响 A,A 正在加载时整图仍
可交互。本能力依赖 graph-traversal-api(/expand);它消费节点、产出更大的图,不直接
触碰 /resolve。

## What Changes

- 新增图画布:把种子节点渲染为初始图(节点 + 边,初始通常只有种子无边)。
- 节点渲染:依据节点的 type / title / subtitle 展示,不同 type 可视觉区分。
- 点击展开:点击节点调用 /expand,把返回的邻居节点与边增量并入当前图。
- 去重:节点按 (type, id) 去重,边按其端点 + 类型去重;重复元素不二次加入。
- 增量布局:新元素加入后重新布局,已有节点位置尽量稳定,避免整图跳动。
- 每节点展开态:加载中 / 错误落在被点节点上;错误可在该节点处重试。
- 已展开标记:节点记录是否已展开,避免对同一节点的无意义重复请求(可选重新展开)。

## Deferred(后续 change)

- 节点详情面板 / 侧边检视。
- 折叠 / 收起子树、隐藏节点。
- 按 type 过滤、图内搜索与高亮。
- 自动布局算法升级(力导向参数调优、分组布局)。
- 撤销 / 重做、导出图、URL 深链。
- 边的方向语义与标签的丰富展示。

## Impact

- Affected specs: 在 `graph-browser` capability 下新增图画布相关需求。
- Affected code: 画布渲染组件、图状态(节点集 / 边集 + 去重)、点击展开调用、
  增量布局、每节点加载/错误态。
- 依赖 graph-traversal-api(/expand)与 browser-shell-and-search 提供的种子接口;
  本能力为只读消费,无数据库交互。
