## 1. 图状态模型

- [x] 1.1 维护节点集与边集;节点键为 (type, id),边键为 (源端点, 目标端点, 类型)
- [x] 1.2 接收来自外壳的种子节点,作为初始图(单节点、通常无边)
- [x] 1.3 提供「并入一批节点与边」的操作,内置去重

## 2. 节点与边渲染

- [x] 2.1 渲染节点,展示 title 与 subtitle
- [x] 2.2 不同 type 的节点在视觉上可区分
- [x] 2.3 渲染边,连接其源 / 目标节点
- [x] 2.4 节点标记是否已展开(用于视觉提示与避免无意义重复请求)

## 3. 点击展开

- [x] 3.1 点击节点时以该节点为入参调用 /expand
- [x] 3.2 把返回的邻居节点与边经去重并入当前图
- [x] 3.3 已存在的节点不重复添加,已存在的边不重复连
- [x] 3.4 展开成功后把该节点标记为已展开

## 4. 增量布局

- [x] 4.1 新元素并入后重新布局
- [x] 4.2 已有节点位置尽量保持稳定,避免整图大幅跳动
- [x] 4.3 新节点放置在被展开节点附近

## 5. 每节点加载与错误态

- [x] 5.1 展开进行中,在被点节点上呈现加载态;其余节点仍可交互
- [x] 5.2 展开失败,在被点节点上呈现错误态并提供重试
- [x] 5.3 同一节点展开进行中,忽略对其重复点击
- [x] 5.4 某节点展开失败不影响其它节点的展开

## 6. 测试

- [x] 6.1 种子节点 → 渲染为初始单节点图
- [x] 6.2 点击节点 → 调用 /expand 并把邻居与边并入图
- [x] 6.3 展开带出已在图中的节点 → 不产生重复节点
- [x] 6.4 展开带出已存在的边 → 不产生重复边
- [x] 6.5 不同 type 节点 → 视觉可区分
- [x] 6.6 展开进行中 → 被点节点显示加载态,整图仍可交互
- [x] 6.7 展开失败 → 被点节点显示错误态与重试,其它节点不受影响
- [x] 6.8 节点展开加载中重复点击 → 仅触发一次 /expand
- [x] 6.9 增量并入后 → 已有节点位置基本稳定,无整图跳动
